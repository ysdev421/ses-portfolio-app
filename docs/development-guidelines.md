# Development Guidelines

## 目的
このドキュメントは、実装・レビュー・デプロイ時に最低限そろえる運用ルールをまとめたものです。

## 文字コード
- すべてのテキストファイルは UTF-8（BOMなし）
- 文字化けが出たファイルは即修正

## 体験版ユーザー
- 体験版ログイン情報は環境変数で管理
- 体験版ユーザーは `role: demo` を強制
- Firestoreルールで書き込み禁止
- UI上もボタン非活性＋理由表示

## Firestoreルール
- `firestore.rules` を更新したら必ずデプロイ
- デプロイ前に `npm run build` で最低限の確認

## SEO
- ページ追加時は以下を確認
  - タイトル / description
  - canonical
  - sitemap対象
  - robots上の制御

## リリース前チェック
- `npm run build` が成功
- 体験版ログインが成功
- 体験版で書き込み不可
- 本番ユーザーで案件追加可能
- 主要導線（LP -> 体験版 -> 登録）が生きている
